RAG · Internal Documentation

Internal-Documentation AI Assistant

In daily use by the team

A chatbot that answers staff questions straight from our internal documentation: runbooks, SOPs, and technical docs with every answer grounded in the source material.

How I built it

  • A retrieval-augmented generation (RAG) pipeline pairing a self-hosted local LLM with a retrieval layer over our internal documentation.
  • Runs entirely on a self-hosted server, so the documentation stays in-house and under our control.
  • Tuned so responses are grounded in the source documents rather than guessed.

Impact

  • In use by the team today — it cut several minutes off the average time to find an answer in our documentation.
  • Gives staff fast, source-grounded answers instead of hunting through files. Adoption, not output, is the measure.

Agent · Custom Tools

Customized AI Agent Deployment

base: Hermes Agent — Nous Research, open source

A self-hosted deployment of an open-source agent framework that I customized and extended for real operational work. The base platform provides the agent runtime, memory, and multi-model support; the value I added is in how I deployed it and the custom tools I built on top.

What I built & configured

  • Deployed it in a containerized backend for sandboxed control, with multi-provider model selection tuned to balance capability against cost.
  • Built a custom infrastructure-monitoring toolset: Prometheus/Grafana metrics with PromQL alerting, systemd service management, hardware/ASIC telemetry (hash rate, temperature, fan speed) with threshold alerts, secure SSH (SOCKS5) tunnels, and log rotation.
  • Built a custom data-investigation toolset: integrations across Dune, Arkham, Etherscan, and The Graph, plus SQL queries on Dune to analyze industry and onchain data.

Extending a capable base agent with custom, API-integrated tools and deploying it in a controlled environment; pointing a general platform at specific, real operational problems.

What these demonstrate

Designing and deploying practical AI systems: both RAG and agentic; building custom tools that integrate external systems via APIs; running models on self-hosted infrastructure; tuning for grounded, reliable output; and, above all, building tools that people actually use.
